Part A: DNS Packet Analysis
1. Start Wireshark and begin packet capture on the active network interface
(WiFi/Ethernet).
2. Open Command Prompt or Terminal and type:
3. nslookup www.google.com
4. In Wireshark, stop the capture after response.
5. Apply filter:
6. dns
7. Select the DNS query and observe the following:
o Transaction ID
o Flags: Query/Response
o Query Name: e.g., www.google.com
o Query Type: A (IPv4)
8. Click on the DNS Response packet and observe:
o Resolved IP Address
o TTL (Time to Live)
o Answer Section
Part B: HTTP Packet Analysis
1. In the browser, type:
2. http://example.com
3. In Wireshark, apply filter:
4. http
5. Observe the HTTP GET request:
o Method: GET
o Host: example.com
o User-Agent
6. Observe the HTTP Response:

o Status Code: 200 OK
o Content-Type: text/html
o Content-Length
